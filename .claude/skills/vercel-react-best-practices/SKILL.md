# React Best Practices

Правила производительности для React 19 SPA (Vite, TanStack Query, Tailwind CSS). Без SSR, RSC и Next.js.

## Устранение водопадов (CRITICAL)

### async-cheap-condition-before-await

Проверяй синхронные условия до await. Если есть дешёвая проверка (null, булев флаг), которая может прервать выполнение — делай её до асинхронного вызова.

### async-defer-await

Заноси await внутрь ветки, где результат реально используется. Если функция принимает промис, но использует его только при каком-то условии — передавай промис и жди его внутри ветки.

### async-parallel

Используй `Promise.all()` для независимых операций. Не делай последовательный await того, что можно запросить параллельно.

## Размер бандла (CRITICAL)

### bundle-barrel-imports

Импортируй напрямую из модуля, а не из баррель-файлов. Баррели убивают tree-shaking.

```tsx
// ПЛОХО: тянет всю библиотеку в бандл
import { Button } from '../components/ui';

// ХОРОШО: прямой импорт
import { Button } from '../components/ui/button';
```

### bundle-conditional

Загружай модули только когда фича активируется. Используй `React.lazy()` или динамический `import()` для кода под флаги, модальные окна или табы вне первого экрана.

### bundle-analyzable-paths

Используй статически анализируемые пути импорта. Динамические шаблонные строки в путях не дают бандлеру предсказать зависимости.

## Оптимизация ререндеров (MEDIUM)

### rerender-defer-reads

Не подписывайся на стейт, который используется только в колбэках. Если значение нужно только в обработчиках событий — читай его через ref.

### rerender-memo

Выноси дорогие вычисления в мемоизированные компоненты. Когда родитель часто ререндерится, а пропсы ребёнка не меняются — оберни ребёнка в `React.memo`.

### rerender-memo-with-default-value

Поднимай не-примитивные значения по умолчанию на уровень модуля. `[]` или `{}` по умолчанию создают новую ссылку на каждом рендере и ломают memo.

### rerender-derived-state

Подписывайся на производный булев признак, а не на сырое значение, которое меняется на каждое нажатие клавиши.

### rerender-derived-state-no-effect

Вычисляй производное состояние на рендере, а не через синхронизацию в `useEffect`. Установка стейта внутри эффекта, зависящего от того же значения, вызывает двойной рендер.

### rerender-split-combined-hooks

Разделяй хуки с независимыми зависимостями. Когда `useMemo` или `useCallback` смешивает часто меняющиеся deps с редко меняющимися — раздели их, чтобы дорогой пересчёт запускался только при изменении реальных входов.

### rerender-move-effect-to-event

Помещай логику взаимодействия прямо в обработчики событий. Если что-то происходит только по клику на кнопку — это место в обработчике клика, а не в `useEffect`, реагирующем на флаг.

### rerender-transitions

Оборачивай несрочные обновления стейта в `startTransition`, чтобы сохранить отзывчивость ввода. Переключение табов, смена фильтров, пересортировка списка — всё, что может подождать кадр.

### rerender-use-deferred-value

Откладывай дорогие рендеры через `useDeferredValue`, когда зависимость обновляется быстро. Отложенное значение запаздывает, позволяя срочному рендеру завершиться первым.

### rerender-use-ref-transient-values

Используй ref для значений, которые часто меняются, но не влияют на вывод рендера (таймстемпы, позиции скролла, кадры анимации).

## Производительность рендеринга (MEDIUM)

### rendering-content-visibility

Используй `content-visibility: auto` для длинных списков. Браузер пропускает рендеринг элементов вне экрана.

### rendering-hoist-jsx

Выноси статический JSX из компонента. JSX, не зависящий от пропсов или стейта, можно определить один раз на уровне модуля.

### rendering-svg-precision

Уменьшай точность SVG-координат. Пути с 6+ знаками после запятой добавляют байты без визуальной разницы.

## Производительность JavaScript (LOW)

### js-request-idle-callback

Откладывай некритичную работу на время простоя браузера через `requestIdleCallback` или `scheduler.postTask`. Аналитика, очистка, прогрев — то, что пользователю не нужно прямо сейчас.

## Продвинутое (LOW)

### advanced-init-once

Инициализируй ресурсы уровня приложения один раз за загрузку страницы, а не при каждом монтировании компонента. Используй модульный lazy-init для сторов, клиентов или тяжёлых синглтонов.
